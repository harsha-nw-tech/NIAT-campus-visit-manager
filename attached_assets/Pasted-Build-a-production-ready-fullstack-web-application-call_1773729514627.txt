Build a **production-ready fullstack web application** called **NIAT Campus Visit Manager**.

This is an **internal tool** used by admins and sales assistants to manage campus visits and walk-in users.

---

# 🧱 TECH STACK (STRICT)

* Frontend: React (Vite)
* Backend: Node.js + Express
* Database: Replit DB (or SQLite)
* State: React hooks
* Auth: Simple token-based (localStorage)
* HTTP: fetch (no axios)

---

# 🔁 CORE APIs

---

## 1. SEARCH USER

POST /api/search-user

Call:

`/api/nw_application/user/phone_number/application/create/v1/`

Return full response including:

* isNewUser
* userId
* applicationId
* student info

---

## 2. GET COMPLETION

POST /api/get-completion

Call:

`/api/nw_application/applications/user_sections_completion/get/v1/`

Return:

```
{
  bookedCampusVisit,
  visitedCampus
}
```

---

## 3. MARK CAMPUS VISITED

POST /api/mark-visited

* Only SALES role allowed

Call:

`/api/nw_application/application/user_section_completion/create_or_update/v1/`

Update BOTH:

* bookedCampusVisit → 100%
* visitedCampus → 100%

Insert log:

```
CAMPUS_MARKED
```

---

## 4. GENERATE DIRECT LINK (UPDATED FLOW)

POST /api/generate-link

Body:

```
{ userId, applicationId, phoneNumber }
```

---

# 🔥 NEW FLOW: TEMPLATE INITIALIZATION (MANDATORY STEP)

When `isNewUser = true`, before generating the direct link:

---

## STEP 1: CALL TEMPLATE RESPONSE API

Backend should call:

```
POST /api/nw_application/application/template_response/update/v1/
```

---

## REQUEST FORMAT

```
{
  "clientKeyDetailsId": process.env.COMMON_DATA_CLIENT_KEY_DETAILS_ID,
  "data": "<stringified JSON>"
}
```

---

## DATA PAYLOAD STRUCTURE

The `data` field must be a stringified JSON string:

```
{
  application_id: applicationId,
  template_id: TEMPLATE_ID,
  section_id: SECTION_ID,
  field_id: FIELD_ID,
  field_value: FIELD_VALUE
}
```

Then stringify like:

```
data: `'${JSON.stringify(payload)}'`
```

# ⚙️ CONFIGURATION

Create:


```
export const TEMPLATE_CONFIG = {

  SECTION_ID: process.env.SECTION_ID,
  FIELD_ID: process.env.FIELD_ID,
  FIELD_VALUE: "PCM / MPC (Maths)"
};
```

---

# 🔌 TEMPLATE SERVICE

Create:

```
// server/services/templateService.js
```

Function:

```
updateTemplate(applicationId)
```

* Uses TEMPLATE_CONFIG
* Calls template API

---

# 🔁 FINAL FLOW IN /api/generate-link

```
1. Receive userId, applicationId, phoneNumber

2. Call updateTemplate(applicationId)

3. If template API fails:
   → return error

4. If success:
   → generate redirectUrl

5. Insert audit log:
   DIRECT_LINK_GENERATED

6. Return redirectUrl
```

---

# 🚫 RULES

* Do NOT generate link if template API fails
* No partial success
* Always log actions
